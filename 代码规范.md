项目采用Python >= 3.11版本开发，使用uv进行依赖管理。

添加依赖时，请使用以下命令：

```bash
uv add <package_name>
```

安装ruff:

```bash
uv tool install ruff
```

## 代码规范
1. 使用PEP 8作为代码风格指南。参阅：https://peps.pythonlang.cn/pep-0008/
2. 使用basic代码检查工具进行静态代码分析。
3. 禁止滥用fallback机制，确保代码的健壮性和可维护性。
4. 所有函数和类必须包含适当的文档字符串，说明其用途和参数。
5. 任何文件开头必须注明该文件的简介。
6. 任何提交commit信息必须清晰描述所做的更改。直接使用ai生成的commit信息是不允许的。
7. src中的所有代码必须在test中有对应的单元测试覆盖，测试覆盖率必须达到100%。
8. 任何函数的参数和返回值必须进行类型注解。
9. 使用ai生成代码时，必须经过严格的人工审核，确保代码质量和安全性。
10. 每个模块在编写完成后，请在examples目录下添加对应的使用示例代码，确保用户能够快速上手使用该模块。
11. 插件代码禁止直接导入其他插件模块；插件间依赖必须通过公开 API、Service、组件签名或协议边界建立，不能写 `from plugins.other_plugin... import ...` 这类跨插件源码依赖。
12. 插件导入自己内部模块时必须使用相对导入，不允许使用 `plugins.my_plugin...` 这种导入自己插件的绝对路径写法。
